Implementation guide

What changed in this redesign and where to apply it in kohit-z/profile-card-shop.

Card layout
  • Avatar moved to cx=108 (was 92) and radius trimmed to 64px — clears room so the name/bio column stops feeling cramped against the circle.
  • Dropped the inline GitHub glyph next to the username; a plain "@handle" in accent color reads just as clearly with less visual noise.
  • Stat, skill, contact and donate tiles now share one badge language: a tinted rounded-square icon slot + bold value + uppercase label, instead of mixed circle/icon treatments.
  • Added hairline dividers between sections so a 4-section card doesn't read as one undifferentiated block.
Effect bugs fixed
  • equalizer and float avatar effects placed bars/shadow at a fixed offset from the avatar center. When the profile section is paired with stats (122px tall vs 214px solo), that offset overshoots the section and draws through the stats row. Fixed by clamping both to frame.y + frame.height.
  • background:aurora and background:matrix duplicate the exact markup in card:aurora / card:matrix — worth extracting to one shared helper both scopes call, so a future tweak doesn't need to land twice.
  • All 24 effects across the four scopes are otherwise functionally complete — every declared name in the registry has a working renderer; nothing was actually a stub.
Builder UI
  • Fields now appear only when their section is active (skills chips hide until "skills" is on, etc.) instead of showing all 14 controls flat regardless of relevance.
  • Contact/donate went from one error-prone comma-separated text input to a per-platform toggle + value row — no more manual platform:value syntax to get wrong.
  • Theme chips became real swatches (gradient + border preview) rather than plain text labels.
  • Panels use CSS grid auto-fit/minmax so the layout reflows on narrow viewports with no explicit breakpoints.
Suggested next steps
  • Add a theme=auto option that emits an SVG <style> block with @media (prefers-color-scheme: dark) swapping CSS custom properties — GitHub already renders README images against the visitor's OS theme, so the card could follow it natively.
  • Drag-to-reorder section chips in the builder, writing the order back into the sections query param (the renderer already supports arbitrary order).
  • Persist builder state in the URL hash so a composed card can be bookmarked/shared before copying the embed.
